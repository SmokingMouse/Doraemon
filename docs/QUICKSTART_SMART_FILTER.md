# 快速验证智能回复过滤功能

## 1. 启动服务

```bash
# 终端 1：启动后端
python main.py

# 终端 2：启动前端
cd frontend-next && npm run dev
```

## 2. 访问界面

打开浏览器访问：http://localhost:3000

## 3. 测试功能

### 登录
- 用户名：`admin`
- 密码：`admin123`

### 发送测试消息

发送一个会触发思考的问题，例如：
```
请解释一下什么是递归，并给出一个简单的例子
```

### 观察效果

**过程中（streaming）**：
- 你会看到 💭 思考过程
- 你会看到 🔧 工具调用
- 内容实时流式展示

**完成后（历史记录）**：
- 只保留最终回复
- 思考和工具信息已被过滤
- 消息历史简洁清晰

## 5. 验证要点

✅ 流式输出包含思考过程（💭）
✅ 流式输出包含工具调用（🔧）
✅ 最终回复已过滤（无 💭 和 🔧）
✅ 消息历史简洁清晰
✅ 用户体验流畅

## 6. 故障排除

### 问题：看不到思考过程
- 确保后端使用了 `--include-partial-messages` 参数
- 检查 WebSocket 连接是否正常

### 问题：最终回复为空
- 查看后端日志，检查提取逻辑
- 运行单元测试验证提取函数

### 问题：前端报错
- 检查 TypeScript 编译：`cd frontend-next && npx tsc --noEmit`
- 查看浏览器控制台错误
